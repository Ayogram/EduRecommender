Hybrid Recommendation Pipeline Architecture

Our recommendation pipeline processes multi-modal student features dynamically. It balances interests (Content-Based) with similar student interactions (Collaborative) and dense dimensionality reductions (Matrix Factorization) using a weighted linear combination scoring.

Neural Recommendation Pipeline Scheme

For deep recommendation tasks, the model integrates user embeddings and item (course) embeddings within a dual Neural Collaborative Filtering (Neural CF) framework:

1. Embeddings Layer Maps discrete User IDs and Course IDs into continuous, dense latent vector embeddings of dimensions k=16 or k=32.
2. Multi-Layer Perceptron (MLP) Concatenates embedding outputs and feeds them through standard dense layers (64 → 32 → 16 units) with ReLU activations to capture highly non-linear interaction rules.
3. Fusion (NeuMF) Combines linear dot product features (Generalized Matrix Factorization) and deep MLP features into a unified dense output, scaling matching logits to continuous probability boundaries.

Software Component Stack

Presentation Layer (Frontend) HTML5, CSS3, Bootstrap 5 templates, custom glassmorphism style rules, and Lucide vector icons.
Business Logic Layer (Backend) Flask WSGI framework handling session authentication, enrollments, classroom sessions, and exams.
AI Model Engine (SVD & CBF) Python Scikit-Learn (TF-IDF vectorizer + cosine similarity), NumPy calculations, and SVD factors.
Database Layer SQLite database storage executing foreign key constraints on relational cascading schemas.
